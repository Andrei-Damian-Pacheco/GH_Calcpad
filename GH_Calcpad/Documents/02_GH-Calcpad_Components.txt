COMPONENTS GH_CALCPAD
=====================

📋 INFORMATION & DIAGNOSTICS
============================

a. GH_Calcpad_Info.cs
Plugin information panel. Displays the plugin version and Calcpad.Core version with license information.
INPUTS:
- (none)
OUTPUTS:
- PluginInfo: Item · string — GH_Calcpad plugin version
- CalcpadInfo: Item · string — Calcpad.Core version and license information

📁 FILE LOADING
===============

b. GH_Calcpad_Load_cpd.cs
Reads .cpd (plain-text) calculation files. Extracts variables, values, and units with automatic file monitoring.
INPUTS:
- FilePath: Item · string — full path to the .cpd file
- CaptureExplicit: Item · bool — if true, only captures explicit format variables (default = false)
OUTPUTS:
- Variables: List · string — names of all variables found
- Values: List · double — numeric values, 1:1 with Variables  
- Units: List · string — units, 1:1 with Variables
- SheetObj: Item · object — CalcpadSheet instance for other components

c. GH_Calcpad_Load_cpdz.cs // Pending development
Reads .cpdz (compiled/zip) calculation files. Supports Base64 decoding and format detection.
INPUTS:
- FilePath: Item · string — full path to the .cpdz file
OUTPUTS:
- Variables: List · string — names of explicit variables found
- Values: List · double — numeric values, 1:1 with Variables
- Units: List · string — units, 1:1 with Variables  
- SheetObj: Item · object — CalcpadSheet instance for other components
- Status: Item · string — file parsing status (TextOK | Binary | NoSource | HtmlParsed)

🔧 VARIABLE MODIFICATION
========================

d. GH_Calcpad_Search_Variables.cs
Filters and modifies specific variables while maintaining complete structure for perfect integration with Play CPD.
INPUTS:
- All Names: List · string — complete list of variable names (from Load CPD)
- All Values: List · double — complete list of variable values (from Load CPD)
- Filter Names: List · string — names of specific variables to modify
- New Values: List · double — new values for filtered variables (1:1 order with Filter Names)
OUTPUTS:
- All Values: List · double — complete array of values with modifications applied → connect to Play CPD
- Modified Names: List · string — names of variables that were modified successfully
- Modified Values: List · double — corresponding modified values (for visualization/debug)
- Not Found: List · string — requested variables that were not found

⚡ EXECUTION & OPTIMIZATION
===========================

e. GH_Calcpad_Play.cs
Main calculation engine. Executes calculations with optional variable override and performance monitoring.
INPUTS:
- Values: List · double — new values for variables (1:1 order with SheetObj Variables), optional
- SheetObj: Item · object — CalcpadSheet from Load components
OUTPUTS:
- ResultEq: List · string — result equations in 'Name=(...)' format
- ResultVal: List · double — numeric results, 1:1 with ResultEq
- Units: List · string — units of final results
- Elapsed: Item · double — calculation time in milliseconds
- Success: Item · bool — true if calculation successful
- UpdatedSheet: Item · object — updated CalcpadSheet for export components

f. GH_Calcpad_Optimizer.cs
AI-powered optimization component with auto-detection capabilities and intelligent caching.
INPUTS:
- Sheet Object: Item · object — CalcpadSheet from Load CPD
- Design Variables: List · string — variables to optimize (auto-detected if empty), optional
- Variable Values: List · double — current values from optimizer (Galapagos input)
- Objective Names: List · string — variables to minimize/maximize (auto-detected if empty), optional
- Optimization Mode: List · string — minimize/maximize/target per objective, optional
- Target Values: List · double — target values for 'target' mode, optional
OUTPUTS:
- Fitness: Item · double — single fitness value for optimizer (lower = better)
- Objective Values: List · double — individual values for each objective
- Status: Item · string — calculation status and convergence information
- Iteration: Item · double — current iteration number
- Best Fitness: Item · double — best fitness found so far
- Suggested Variables: List · string — auto-detected variables for optimization
- Convergence Info: Item · string — progress analysis and recommendations

🔍 RESULT FILTERING
===================

g. GH_Calcpad_Search_Results.cs
Filters specific results from calculation outputs for focused post-calculation analysis.
INPUTS:
- Result Equations: List · string — complete list of result equations (from Play CPD)
- Result Values: List · double — complete list of result values (from Play CPD)
- Filter Names: List · string — specific variable names to extract from results
OUTPUTS:
- Filtered Equations: List · string — equations that match Filter Names
- Filtered Values: List · double — values corresponding to filtered equations
- Found Names: List · string — variables that were successfully found
- Not Found: List · string — requested variables not found in results

💾 SAVING & EXPORT
==================

h. GH_Calcpad_Save_cpd.cs
Saves modified CPD/TXT files with controlled execution and flexible naming.
INPUTS:
- Updated Sheet: Item · object — CalcpadSheet from Play CPD with modifications
- File: Item · string — base name (without extension), optional
- Output Folder: Item · string — destination folder path, optional
- CPD/TXT: Item · bool — false = .cpd (default), true = .txt, optional
- Execute: Item · bool — true = save (default = false)
OUTPUTS:
- Save Path: Item · string — complete path of the saved file
- Success: Item · bool — true if file was saved successfully

i. GH_Calcpad_Export_html.cs
Exports HTML reports using Calcpad's native Convert engine with controlled execution.
INPUTS:
- Updated Sheet: Item · object — calculated CalcpadSheet (from Play CPD)
- File: Item · string — base name (without extension), optional
- Output Folder: Item · string — destination folder, optional
- Execute: Item · bool — true = export (default = false)
OUTPUTS:
- HTML Path: Item · string — path of the generated .html file
- Success: Item · bool — true if .html was generated

j. GH_Calcpad_Export_pdf.cs
Generates professional PDF reports using Calcpad's native export engine with controlled execution.
INPUTS:
- Updated Sheet: Item · object — calculated CalcpadSheet (from Play CPD)
- File: Item · string — base name (without extension), optional
- Output Folder: Item · string — destination folder, optional
- Execute: Item · bool — true = export (default = false)
OUTPUTS:
- PDF Path: Item · string — path of the generated PDF file
- Success: Item · bool — true if PDF was generated

k. GH_Calcpad_Export_word.cs
Exports editable Word documents using multiple conversion methods with controlled execution.
INPUTS:
- Updated Sheet: Item · object — calculated CalcpadSheet (from Play CPD)
- File: Item · string — base name (without extension), optional
- Output Folder: Item · string — destination folder, optional
- Execute: Item · bool — true = export (default = false)
OUTPUTS:
- Word Path: Item · string — path of the generated .docx file
- Success: Item · bool — true if .docx was generated

WORKFLOW INTEGRATION
====================

BASIC WORKFLOW:
Load CPD → Play → Export

ADVANCED WORKFLOW:
Load CPD → Search Variables → Play → Search Results → Save/Export

OPTIMIZATION WORKFLOW:
Load CPD → Optimizer → Galapagos → Save/Export

COMPLETE PARAMETRIC WORKFLOW:
Load CPD → Search Variables → Play → Search Results → Save CPD → Export PDF/Word
